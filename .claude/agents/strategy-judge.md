# 確定事実(裁定の物差し)

1. **スコア**: score(task) = max(1, 25 − ln(memory + params))。memory = 中間テンソル(ノード出力 + value_info、'input'/'output' 名は除外)の bytes 合計。initializer は params 側。単一ノードグラフは memory 0。詳細: `proposals/001-zero-risk-onnx-cost-reduction.md`
2. **正しさは全か無か**: train + test + arc-gen 全例 + **非公開プライベートセット**で全セル一致が必要
3. **事故 1(-15.4 点)**: 公開例 265 件にフィットした決定木ネットはローカル全合格でも非公開セットで 0 点(task096)。**例フィットは禁止、仕様(inputs/arc-gen-repo のジェネレータ)のコンパイルのみ可**。詳細: memory/private-set-overfit.md
4. **事故 2(-8.6 点)**: macOS と Linux の浮動小数点差でローカル検証は公式と乖離し得る(±両方向)。ローカル合格だけを根拠にグレーダー実証済みネットを置き換えてはならない。マージン(生出力に 0〜0.25 の際どい値がない)が防壁
5. **検証ゲート**: `scripts/golf/try_candidate.py` が唯一の昇格経路。gold 全例一致 + マージン + 現職より厳密に安い
6. **ln スケール**: コスト半減 = +0.69。数 % の削減はスコアに出ない。構造的削減(桁)だけが意味を持つ
7. **時間制約**: 2026-06-13 12:00 までに公式 6550。1 時間あたり +11.5 が必要ライン
8. **禁止 op**: Loop/Scan/NonZero/Unique/Script/Function/Compress/Sequence 系。静的形状必須。1.44MB/ファイル
9. task054/158 はグレーダー処理エラーの前歴(別系統 zip)。差し替えは特別審査

# 裁定手順

1. アイデアを 1 文で再述する(誤解があれば指摘)
2. 上の事実と矛盾しないか照合。矛盾があれば**却下 + どの事実に反するか**
3. 期待値を概算: 影響タスク数 × 平均 Δln(cost)。時間コスト(実装+検証)との比
4. リスク分類: ゼロリスク(数値不変)/低(マージン保護内)/高(非公開セット・グレーダー差の懸念)
5. 判定: **採用 / 条件付き採用(条件を明記)/ 却下**。迷ったら「小さく実験して実測」を提案する

# provenance 監査(昇格ネットのレビュー時)

scratch の REPORT.md と builder スクリプトを読み、以下があれば**差し戻し**:
- 例データから機械学習・フィッティングした重み/木/テーブル
- 可視例の座標・色・個数への直接依存(ジェネレータ仕様が保証しない範囲)
- 正当化のない unroll 深さ・クロップ寸法(ジェネレータ分布 or 現職の実績深度による根拠が必要)

# 出力形式

**判定**(採用/条件付き/却下)を最初の行に。次に根拠(事実番号を引用)、期待値概算、推奨アクション。簡潔に。

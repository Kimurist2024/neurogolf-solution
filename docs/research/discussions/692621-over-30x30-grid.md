## Body

Hi, I have a question about the NeuroGolf dataset constraints.

The competition description says grids are at most 30x30, and the submitted ONNX models also use a fixed input shape of `[1, 10, 30, 30]`.
However, when checking some local `task*.json` files, I found examples whose input or output sizes appear to exceed 30x30.

In that case, how should we handle them?

My current understanding is that examples larger than 30x30 cannot be represented by the competition input format itself, so they would seem to be out of scope for submitted models.
Should such examples be treated as invalid / ignored, or am I misunderstanding something about the intended preprocessing?

Also, are the official Kaggle evaluation files guaranteed to contain only examples with input/output sizes up to 30x30?

Thanks in advance.

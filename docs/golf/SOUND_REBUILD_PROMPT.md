# NeuroGolf: 健全な安ネット再構築プロンプト(汎用)

> 目的: 「攻めて安くする」のではなく **真ルールを正しく実装して安くする**。公開例に焼き込んだルックアップ型は private セットで必ず黒(private-0)になる。このプロンプトは黒タスク解析(101/112/134/168/192/198/201/219/343/344/391)から抽出した方法論。

## 0. 大前提 — なぜ安ネットは黒になるか

黒ネットの共通構造(**これが出たら過学習を疑え**):
- `ArgMax + Gather + ScatterElements / ScatterND / GatherND`(入出力の**ルックアップ暗記**)
- `TfIdfVectorizer`(×8〜36)/ `Hardmax` / 変則 `Einsum`(**op 悪用でパターン一致を暗記**)
- 本質的に反復/連結成分が要るのに**極少ノードで圧縮**(例 task198 の flood-fill を Einsum×2+Hardmax の3ノード)
- `AffineGrid` / `Resize` / `CenterCropPad` で**幾何変換を近似**(サイズ変動で破綻)

根本原因: **出力を決めるパラメータ(色の役割・軸位置・倍率・テンプレ形状・周期・並び順)が入力依存の大域量**。公開例に焼いた固定写像は、非公開の別インスタンス(別サイズ・別位置・別色・別個数)で必ず外れる。arc-gen 262件を通っても private では黒。

## 1. 必須ワークフロー(この順を守る)

1. **真ルールを解読する(例フィットではなく)**
   - `inputs/sakana-gcg-2025/raw/taskNNN.py` の難読 `p`(lambda/def)が**検証済みの正解変換関数**。1文字変数・複素数座標・内包表記を丁寧に展開して**人間可読な擬似コード**に落とす。
   - `inputs/neurogolf-2026/taskNNN.json` の train+test+arc-gen 全ペアで `p` を実行し **100% 一致**を確認(解読の正しさの証明)。

2. **ルールを分類する(下記 Type A〜D)** → 安く正しく作れるか判断。

3. **Type A/B なら実装、C/D は慎重 or 現職維持。**

4. **fresh 入力で汎化検証(必須)**
   - 真ルールで**新規入力**(別サイズ・別位置・別個数・別色)を合成し、作ったネットと真ルール出力を比較。**100% でなければ過学習 → 採用禁止**(arc-gen だけの gold は不十分)。
   - 例: task192 の安ネットは fresh で 26% 不一致 → 黒。局所ルールを正しく実装したネットは fresh 0% になる。

## 2. ルール分類 — 安く正しく作れるか

| Type | 性質 | 例 | 実装方針 | 安価健全化 |
|---|---|---|---|---|
| **A 局所** | 出力セル = k×k 近傍 (+ 1個の大域色選択) で決まる | **192**(argmax色+H/V隣接デノイズ)、**344**(隣接2,3ペア再彩色) | 小 Conv(plus型/3×3)で厳密実装 | **◎ 最優先。現職が高コストなら大幅減の白ネットが狙える** |
| **B 有界反復** | 局所検出 + **有界長の伝播**(grid≤30なので有限段) | **168**(L字検出→対角レイ≤9段)、343(周期タイル) | 検出Conv + **固定段の方向シフト+OR** をアンロール | ○ 構造を正しく組めば健全。段数はグリッド上限で固定 |
| **C 大域トポロジー** | 連結成分/flood-fill/テンプレマッチが必須 | **198**(0連結成分の矩形性→3/4)、**101**(CCテンプレ拡大スタンプ)、**219**(オブジェクト相関整列) | ラベル伝播を最大直径(~60)段アンロール+成分reduce。**構造的に高コスト** | **✗ 安く作れない。現職(高コスト・正しい)を維持。安い版は必ず黒** |
| **D 大域並べ替え/幾何** | ヒストグラム+argsort / データ依存クロップ / ピボット鏡映 | **391**(頻度で希少3色)、**134**(役割判定→縮小)、**112**(4回鏡映)、**201**(枠内スタンプ+反転) | ReduceSum+TopK/ArgMax+Gather。ただし下記地雷に注意 | △ 実装可だが地雷多い。tie/dtype/データ依存gatherを厳密に |

**判断のコツ**: 現職コストが「ルールの構造的複雑さ」に対して**不自然に安い**なら、それは暗記型 = 黒候補。逆に Type A で現職が高コスト(例 task192=1609, task344=401)なら、**正しい局所実装で大幅に安く&白**にできる大チャンス。

## 3. タスク別・私的0点の地雷(再実装でも踏むもの)

- **タイブレークの非決定性(task391)**: 真ルールが同数時のタイブレークを Python の `{*A}` 集合反復順(ハッシュ順=事実上不定)で決める場合、private でタイが起きると**決定的な (count, 色番号) 実装は必ず食い違う**。→ この種は再実装でも直せない(真ルール自体が ill-defined)。**深追いせず現職維持**。
- **uint8 の TopK はグレーダーで死ぬ**(既知)。カウント/ランキングは **f32** で扱う。
- **Conv/ConvTranspose の bias長 < 出力ch** = ±20 の非決定フリップ。提出前 `python scripts/golf/check_conv_bias.py <zip>` = 0 件を必ず確認。
- **出力エンコード**: 正解色 k のセル → ch k =1。グリッド内の背景(色0)→ ch0 =1。**グリッド外(境界外)→ 全ch 0**(zero-hot)。`NZ=ch1..9の和`、`IG=全chの和` で「グリッド内/外」と「非ゼロ/背景」を区別せよ(`1−ch0` は境界外を誤判定する)。
- **周期/倍率/軸の決め打ち禁止**(task343 の L=6/8、task134 の倍率C、task112 のピボット)。データ依存量は必ずグリッドから算出。

## 4. コストモデルの実務(安く保つ鍵)

- **入力 `[1,10,30,30]` と出力テンソルは cost 対象外**(scorer が `input`/`output` を除外)。→ 中間テンソルだけが params+memory に効く。
- **中間は `[1,1,30,30]`(uint8 化で 900B)に抑える**。`[1,10,30,30]` の中間(=36KB)を作らない。最終出力の組み立て `[1,10,30,30]` は「output」テンソルなので無料。
- opset≥14 で uint8 算術可。bool/uint8 マスクを活用。
- コストは ORT 実行せず静的計算可(`infer_shapes strict_mode` + params)。ただし value_info 未解決だと過小評価するので、**採用判定は gold 検証の実コスト**で。

## 5. 実行優先ターゲット(このプロンプトで狙うべき黒→白化)

1. **task192(現職1609, Type A 局所)** — argmax色A + 「A が縦±1 かつ 横±1 に隣接する非ゼロセルを A に」の3×3クロス。小 Conv2本(1×3, 3×1)で厳密実装 → 数百コスト級の白が狙える(**+0.4〜0.9**)。
2. **task344(現職401, Type A 局所・回転はダミー)** — 「2の隣に3→8」「3の隣に2→0」の同時局所書き換え。plus型 Conv で隣接判定 → 現職より安い白の余地。
3. **task168(現職432, Type B 有界)** — L字検出 + ≤9段の対角シフト。正しく組めば健全。
4. **task343(現職~172, Type B)** — 周期4判定 → L=6/8 タイル。小さいがマージンあり。

**避けるべき(Type C/D の黒)**: 198/101/201/219/391/134/112 は構造的に安く作れない or tie地雷。現職(高コストでも白)を維持し、安い候補が来ても **fresh 検証 100% 未満なら採用しない**。

## 6. コパイロット(codex/kimi)への依頼テンプレ

```
task{NNN} の健全な安ネットを作れ。
1. inputs/sakana-gcg-2025/raw/task{NNN}.py の真ルールを擬似コードに解読し、
   inputs/neurogolf-2026/task{NNN}.json 全ペアで100%一致を確認して報告。
2. ルールを Type A(局所)/B(有界反復)/C(大域トポロジー)/D(大域並べ替え)に分類。
   C なら「安く作れない、現職維持」と報告して終了。
3. A/B/D なら禁止op(Loop/Scan/NonZero/Unique/Script/Function)無し・静的形状で実装。
   ルックアップ暗記(ArgMax+Gather+Scatter で出力表を引く)・TfIdfVectorizer・Hardmax は禁止。
   中間は[1,1,30,30]/uint8 に抑える。TopKはf32。bias長=出力ch。
4. 真ルールで fresh 入力(別サイズ/位置/個数/色)を100件合成し、ネット出力と比較。
   100%一致でなければ過学習として作り直す。1件でも外れたら報告して中断。
5. gold(train+test+arc-gen 完全一致)+ fresh 100% + check_conv_bias=0 を満たしたら
   コストと共に artifacts/handcrafted/task{NNN}.onnx に出力。
```

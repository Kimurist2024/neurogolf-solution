## 運転ルール(各ウェーブ完了時に Fable が実行)

1. ワーカー結果収集 → `merge_external.py --tasks all --zip --source X=artifacts/handcrafted`
2. ローカル期待スコア改善があれば**即提出**(1 タスクでも)
3. 公式スコア確認 → ローカル予測と乖離したら原因調査(過適合・Error の疑い)を最優先
4. `current_costs.json` を再生成 → 次ウェーブのキュー(cost 降順、直前ウェーブで停滞したタスクは 1 回スキップ後に失敗ログ付きで再投入可)
5. 次ウェーブ投入: ワーカー 10〜16 体、**仕様コンパイル限定プロトコル**(proposal 003 Amendment 1)
6. 累計 +150 達成、またはキューに cost>2000 のタスクが尽きたら終了

## 安全規程(変更不可)

- 例フィット禁止 / provenance 宣言義務(task096 事故の再発防止)
- task054・158 は差し替え時に特別審査(Error 前歴)
- 提出はゲート通過分のみ。事故時は当該タスクだけグレーダー実証版に戻して即再提出
